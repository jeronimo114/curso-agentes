Seguridad

El código que genera la IA
no es seguro por default

Funciona. Pero expone datos, gasta tu dinero
y puede comprometer a tus usuarios.

Seguridad
Las 5 reglas que no puedes ignorar Obligatorio
  • 🚫 Nunca subas un archivo .env a GitHub. Ahí van tus API keys, passwords de DB, y secretos. Si se filtran, alguien puede vaciar tu cuenta de AWS en horas.
  • 🔑 Variables de entorno, siempre. En Vercel van en "Environment Variables". En desarrollo, en un archivo .env.local que nunca commiteas.
  • 👤 No uses datos reales de personas en demos. Crea datos ficticios para probar. GDPR existe y aplica aunque seas indie.
  • 🔒 Habilita Row Level Security en Supabase. Sin RLS, cualquier usuario puede leer los datos de otros. La IA a veces no lo activa sola.
  • 💸 Pon límites de gasto en las APIs de IA. OpenAI, Anthropic y similares cobran por uso. Un loop infinito en producción puede costarte cientos de dólares.

Desarrollo
Cómo atacar las 2h 45min Guía sugerida

1 Definir (15 min)

¿Qué problema resuelve tu app? ¿Quién la usa? Escríbelo en 2 líneas antes de abrir cualquier herramienta.

2 Primer prompt (20 min)

Describe tu app completa al agente. Incluye: nombre, features principales, diseño y tech stack.

Crea una app llamada [nombre] que [descripción]. Features: [lista]. Stack: React + Supabase + Vercel.

3 Iterar (1h 30min)

Un feature a la vez. Si algo falla, describe el error exacto al agente. No borres todo por un bug.

4 Deploy (30 min)

Subir a GitHub → conectar Vercel → agregar env vars → URL en vivo. Antes del break.

⚠ Error más común

Querer hacer todo. Elige 1–2 features que funcionen bien. Una app simple que funciona vale más que una compleja rota.
