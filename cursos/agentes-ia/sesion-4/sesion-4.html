Seguridad

Seguridad · 01
Regla 01

Nunca subas
un .env a GitHub

Tus API keys, passwords y secretos viven ahi. Si se filtran, alguien puede vaciar tu cuenta de AWS en minutos.

Seguridad · 02
Regla 02

Variables de entorno,
siempre

En Vercel van en Environment Variables. En local, en un archivo .env.local que jamas commiteas al repositorio.

Seguridad · 03
Regla 03

Sin datos reales
en demos

Usa datos ficticios para probar. GDPR existe y aplica aunque seas indie y aunque tu mercado sea Colombia.

Seguridad · 04
Regla 04

Activa Row
Level Security

Sin RLS en Supabase, cualquier usuario autenticado puede leer los datos de todos los demas. La IA no lo activa por defecto.

Seguridad · 05
Regla 05

Pon limites de gasto
en las APIs

OpenAI, Anthropic y similares cobran por uso. Un loop en produccion puede costarte cientos de dolares en horas. Configura billing alerts desde el dia uno.

1 vercel.com — Sign up con GitHub. Gratis para proyectos personales sin limite de tiempo.
2 Importar repositorio de GitHub. Cada push a main despliega automaticamente.
3 Environment Variables — tus keys de Supabase van aqui, nunca en el codigo fuente.
4 Tu app queda en tu-app.vercel.app — URL real, SSL, CDN global.
100 GB bandwidth / mes  ·  SSL automatico  ·  CDN global

1 supabase.com — New Project. PostgreSQL listo en 2 minutos.

3 Settings → API → copiar URL y anon key → pegar en Vercel como env vars.

Desarrollo

Desarrollo
1h 30min para construir tu app 1:30 — 3:00

1 Definir (10 min)

Que problema resuelve tu app. Quien la usa. Escribelo en 2 lineas antes de abrir cualquier herramienta.

2 Primer prompt (15 min)

Describe tu app completa al agente. Nombre, features principales, diseno y tech stack en un solo mensaje largo.

Crea una app llamada [nombre] que [descripcion]. Stack: React + Supabase + Vercel.

3 Iterar (50 min)

Un feature a la vez. Si algo falla, describe el error exacto al agente. No borres todo por un bug.

4 Deploy (15 min)

Subir a GitHub, conectar Vercel, agregar env vars. Necesitas una URL real antes de las 3:00.

Error mas comun: Querer hacer todo. Elige 1 o 2 features que funcionen bien. Una app simple que funciona vale mas que una compleja rota.
