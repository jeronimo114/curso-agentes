Sesión práctica · Agentes de programación

Codex en acción para crear, iterar y publicar más rápido.

Resumen de la sesión: flujo de trabajo, herramientas recomendadas, riesgos y mitigaciones para uso profesional.

Qué vimos en la sesión

  • Prompting guiado: instrucciones directas, claras y accionables.
  • Markdown como base: contexto personal, estilo visual y trazabilidad.
  • Generación de entregables: HTML, PDF y sitios web personales.
  • Deploy real: GitHub + Vercel para publicar en internet.
  • Iteración rápida: captura de errores, screenshot y corrección inmediata.

Flujo recomendado de trabajo

  • 1. Crear archivo de contexto contexto-personal.md con perfil, tono, objetivos y datos reutilizables.
  • 2. Crear archivo de estilo estilos-presentacion.md con lineamientos visuales.
  • 3. Pedir entregable “Con base en estos archivos, crea presentación/sitio en HTML”.
  • 4. Iterar por bloques una conversación por feature para cuidar ventana de contexto.
  • 5. Publicar y compartir push a GitHub y deploy en Vercel.

Aprendizajes clave

  • Markdown mejora calidad de respuesta y portabilidad entre modelos.
  • La precisión del prompt define la calidad del resultado.
  • Los agentes ejecutan tareas completas, no solo responden preguntas.
  • La trazabilidad evita pérdida de contexto y retrabajo.
  • El valor está en el criterio humano: decidir, validar y priorizar.

Riesgos reales al usar agentes

  • Alucinaciones: respuestas plausibles pero incorrectas.
  • Fuga de datos: exponer credenciales o información sensible.
  • Dependencia excesiva: aceptar código sin entenderlo.
  • Deuda técnica: iterar rápido sin arquitectura mínima.
  • Contexto degradado: conversaciones largas que pierden precisión.

Cómo mitigarlos desde hoy

  • Validar todo con pruebas, logs y revisión manual crítica.
  • Separar contextos por proyecto y por feature.
  • No compartir secretos sin vault, .env y controles de acceso.
  • Versionar cambios en Git con commits pequeños y claros.
  • Estandarizar prompts y mantener markdowns de referencia actualizados.

Plantilla mínima de archivos

  • 00_contexto_personal.md perfil, expertise, tono, objetivos.
  • 01_estilo_visual.md tipografía, color, estructura de slides.
  • 02_estado_proyecto.md decisiones, bugs, pendientes, próximos pasos.
  • 03_prompt_base.md instrucciones reutilizables para generación.

Resultado: mayor consistencia, menos alucinación y mejor velocidad de entrega.
